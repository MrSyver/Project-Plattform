Funktionen

Alles, was ein professionelles BI-Portal braucht

Modular aufgebaut: Bausteine je Projekt zusammenstellen, ohne für jeden Kunden neu zu entwickeln.

📊

Power BI mit SSO

Reports per Auswahl oder Link einbinden. Bei AD-Login automatisches Single-Sign-On – die Berechtigungen (RLS) des Nutzers greifen. Niemals „publish to web".

🗂️

Projekträume

Getrennte Bereiche mit eigenem Zugriff – frei kombinierbar aus AD-Gruppen, einzelnen Nutzern und externen Gästen ohne AD-Konto.

📎

Dateiaustausch

Dateibibliothek je Projekt mit Ordnern, Versionierung und Pflicht-Virenscan – jeder Zugriff wird protokolliert.

🧩

In-App-Editor

Editoren bauen Seiten im WYSIWYG-Modus aus Blöcken zusammen: Text, BI-Report, Dateiaustausch, Video. Entwurf → Veröffentlichen.

🔐

Enterprise-Security

Entra ID / AD, VPN-/Netzwerk-Isolation, Secrets im Vault, lückenloses Audit-Log. Auf DORA/BAIT/ISO 27001 ausgelegt.

☁️

On-Prem & Cloud

Ein Docker-Image, zwei Wege: komplett On-Prem (air-gapped-fähig) oder Azure – nur die Konfiguration unterscheidet sich.

Betrieb

Zwei vollständige Deployment-Pfade

Gleiches Image, austauschbare Adapter je Schicht (Ports & Adapters). Je Kunde eine isolierte Instanz.

☁️ Komplett Cloud (Azure)

  • Azure Container Apps + Azure SQL
  • Entra ID (inkl. B2B-Gäste)
  • Key Vault · Blob Storage
  • Power BI Service (Embedded)
  • Private Endpoint + VPN/ExpressRoute

🏢 Komplett On-Prem

  • Docker Compose + SQL Server
  • AD FS / Keycloak gegen lokales AD
  • HashiCorp Vault · Volume/MinIO
  • Power BI Report Server
  • Interner Zugang im Firmennetz/VPN
Anleitung

In wenigen Minuten lokal starten

Voraussetzung: .NET 8 SDK. (Für den Container-Betrieb zusätzlich Docker.)

Repository holen 
# klonen …
git clone https://github.com/MrSyver/Project-Plattform.git
cd Project-Plattform
Bauen & testen 
dotnet build ReportingPlattform.sln
dotnet test  ReportingPlattform.sln   # alle Tests grün
Lokal starten 
dotnet run --project src/ReportingPlattform.Web
# Health-Check: http://localhost:5080/healthz
Oder als Container (On-Prem) 
cp .env.example .env      # Werte setzen
docker compose up -d     # App + SQL Server + ClamAV
